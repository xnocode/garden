Parkinson's Law of Triviality — better known as bikeshedding — says that a committee will spend more time arguing about the color of a bike shed than about the design of a nuclear reactor, because everyone has an opinion on color and almost no one has an opinion on reactors.

Digital gardens have the same failure mode. ==It is much easier to reorganize your notes than to write them.== It is much easier to tweak your theme than to think. It is much easier to install a new plugin than to actually capture an idea before it escapes.

> [!warning] The trap
> The garden *feels* productive when you're tending it. You're moving files, fixing tags, adjusting CSS, syncing configs. None of it is writing. All of it feels like writing.

There's a spectrum here. On one end: never tending, letting the garden rot. On the other: only tending, never planting. The productive gardener is somewhere in the middle — mostly planting, occasionally weeding, rarely redesigning the beds.

A few rules I try to follow:

1. **Write the note before you file it.** If you catch yourself wondering which folder a note should go in before you've written it, stop. Write first; file later.
2. **Tags are for finding, not for organizing.** Don't build a tag ontology. Just tag with the words you'd search for.
3. **Links beat folders.** A note can be in one folder. It can link to a hundred others. Links are the denser medium.
4. **The graph is a diagnostic, not a goal.** If your graph is sparse, you need more links. If it's a hairball, you need fewer. Don't optimize it for beauty.

> [!tip]
> Set a timer. Ten minutes of tending per session, max. When the timer goes off, you're either writing or you're done.

The garden is a tool for thinking. The moment the tool becomes the task, you've bikeshedded yourself out of the actual work.
