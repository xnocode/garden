Money compounds because the interest earns interest. Knowledge can compound the same way — but only if you let it. A note that connects to three other notes is worth more than three notes that connect to nothing, because the next note you write can link to all four, and the one after that can build on the structure that's emerging.

The catch: compounding only works if the principal stays invested. A note in your head doesn't compound. A note in a notebook you never open doesn't compound. A note in a system you've abandoned doesn't compound. ==The note has to stay reachable — by you, and by the other notes.==

> [!tip] The two requirements
> For knowledge to compound, your notes need two things:
> 1. **Permanence** — they have to survive long enough to be found again.
> 2. **Connection** — they have to link to other notes, so the structure can grow.

This is why the [[Zettelkasten Method]] works. Atomic notes, uniquely addressed, densely linked. Each new note is a new node in a graph that gets denser — and more valuable — over time. The first hundred notes feel slow. The next hundred feel easier. By the thousandth note, you're not writing from scratch anymore; you're assembling from a library that's been growing under you for years.

The same logic applies to [[Building a Second Brain]]. The CODE method (Capture, Organize, Distill, Express) is really about keeping the principal invested: capture so nothing escapes, organize so things are findable, distill so the signal strengthens, express so the structure gets tested against reality.

> [!warning]
> The enemy of compounding is not forgetting — it's reorganizing. Every time you rebuild your system from scratch, you reset the compounding. The connections you'd built don't transfer; you have to discover them again.

So: pick a system. Stick with it. Let the notes accumulate. Let the links thicken. Don't optimize the structure before the structure has had time to teach you what it wants to become.
